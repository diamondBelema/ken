Install

Download binary

Pre-built binaries for Linux and Windows:

PlatformDownload
Linux (x86_64) ken-linux-amd64
Windows (x86_64) ken-windows-amd64.exe 
# Linux
curl -LO https://github.com/diamondBelema/ken/releases/latest/download/ken-linux-amd64
chmod +x ken-linux-amd64 && mv ken-linux-amd64 ~/.local/bin/ken

From source

git clone https://github.com/diamondBelema/ken.git
cd ken
go build -o ken ./cmd/ken

Requires Go 1.21+. Move the binary somewhere on your $PATH:

mv ken ~/.local/bin/

Run directly

go run ./cmd/ken

The learning model

ken implements a three-layer learning system based on cognitive science research on desirable difficulties and self-explanation. Each layer builds on the previous one, creating a progression from passive exposure to active recall.

Layer 1: Familiarity

Command: ken map <subject>

Walk through the concept tree and read summaries. No typing, no pressure — just exposure. This builds initial familiarity with the material. Concepts you've expanded are marked as "seen" in your progress.

Supports course group filtering (--group bioenergetics) so you can focus on one section of a course at a time.

Layer 2: Reflection

Command: ken reflect <subject>

For each concept, type your own explanation before seeing the canonical answer. This leverages the self-explanation effect — generating your own answer before checking strengthens retention more than passive re-reading.

Reflections are saved as tagged notes you can review later. The system tracks how many times you've reflected on each concept and when.

Layer 3: Mastery

Commands: ken flashcards <subject>, ken quiz <subject>

Active recall under pressure. Flashcards use 5-level grading (Unknown → Mastered). Quizzes support multiple-choice, true/false, and fill-in-the-blank. Both feed a Bayesian confidence engine with time-decay that tracks what you actually know.

Learning flow

The recommended progression for a new topic:

ken map biochemistry          # 1. Familiarize — see the concept tree
ken reflect biochemistry      # 2. Reflect — type your explanations
ken read biochemistry         # 3. Deep read — engage with lecture notes
ken flashcards biochemistry   # 4. Master — active recall, 5-level grading
ken quiz biochemistry         # 5. Test — quiz under pressure

You don't have to follow this order strictly — ken doesn't enforce it — but cognitive science research consistently shows that interleaving exposure, self-explanation, and active recall produces stronger long-term retention than any single method alone.

Progress model

Progress is stored per-layer, per-concept in ~/.local/share/ken/<subject>.json:

{
  "concepts": {
    "c-glycolysis": {
      "familiarity": { "seen": true },
      "reflection":  { "count": 3, "last_at": 1721366400 },
      "mastery":     { "confidence": 0.72, "last_reviewed_at": 1721366400 }
    }
  }
}

Content at ~/Documents/learn/subjects/ stays read-only. Your git repo stays clean. Progress lives in your state directory.

Content format

Directory structure

~/Documents/learn/subjects/
  biochemistry/
    groups.yaml                    # optional: course group divisions
    concepts/
      glycolysis.md
    flashcards/
      glycolysis.md
    quizzes/
      glycolysis.md
    notes/
      lecture-01.md                # raw readable content

Course groups

Optional file at the subject root that defines structural divisions (e.g. textbook chapters, lecture weeks):

groups.yaml
groups:
  - id: bioenergetics
    name: Bioenergetics
    concepts:
      - c-atp
      - c-glycolysis
      - c-krebs
  - id: enzymes
    name: Enzymes
    concepts:
      - c-enzyme-kinetics
      - c-inhibition

Groups are used by ken map --group <id> to filter the concept tree, and appear as sections in the dashboard progress display.

Concept file

concepts/glycolysis.md
---
format_version: 1
type: concept_set
set: Glycolysis
concepts:
  - id: c-glycolysis
    name: Glycolysis
    parent_id: null
    summary: "Metabolic pathway that breaks down glucose into pyruvate"
    diagrams:
      - type: external
        src: glycolysis-steps.svg
  - id: c-pfk1
    name: Phosphofructokinase-1
    parent_id: c-glycolysis
    summary: "Rate-limiting enzyme of glycolysis"
---

## c-glycolysis
The metabolic pathway that breaks down glucose into pyruvate, yielding a net 2 ATP.

## c-pfk1
Rate-limiting enzyme of glycolysis. Inhibited by ATP and citrate.

questions:
  - id: bch-q001
    concept_id: c-pfk1
    type: mcq
    question: "Which enzyme is rate-limiting?"
    options:
      - "Hexokinase"
      - "PFK-1"
      - "Pyruvate kinase"
    answer: 2
    explanation: "PFK-1 is the rate-limiting enzyme."
  - id: bch-q002
    type: true_false
    question: "Glycolysis yields 4 ATP net"
    answer: false
    explanation: "Net yield is 2 ATP (4 produced - 2 invested)."
---

Notes and concept tagging

Notes in the notes/ directory are plain markdown. ken parses them for concept references using [c-concept-id] tags, which enable concept hopping in ken read:

notes/lecture-01.md
# Glycolysis [c-glycolysis]

Glycolysis is the first step in cellular respiration.

## The PFK-1 Step [c-pfk1]

Phosphofructokinase-1 is the key regulatory enzyme.
ATP and citrate inhibit PFK-1 [c-pfk1].

Heading tags create tree nodes in the concept hop sidebar. Inline tags create highlight markers in the text. Both are optional — notes work fine without any tags.

For the full spec — field definitions, YAML quoting rules, and templates — see CONTENT-CREATION.md.

CLI reference

Core commands

Command Description
ken Launch the dashboard — see all subjects at a glance
ken map <subject> Familiarity layer — walk the concept tree and read summaries
ken reflect <subject> [concept-id] Reflection layer — type your explanation, then see the canonical answer
ken read <subject> Deep reading — browse lecture notes with concept hopping
ken flashcards <subject> Mastery layer — study flashcards with 5-level grading
ken quiz <subject> Mastery layer — quiz with mcq, true/false, and fill-in-the-blank
ken notes <subject> View and manage user notes
ken stats View detailed stats and confidence trends
ken lint [subject] Validate content files (exits 1 on errors)
ken version Print version and platform
ken self-update Update ken to the latest release

ken lint accepts --json for machine-readable output. ken map accepts --group <id> to filter by course group. ken reflect accepts an optional concept ID to start at a specific concept.

Registry commands

Command Description
ken search <query> Search the package registry
ken add <author/package> Install a study package from the registry
ken list List installed packages
ken remove <author/package> Uninstall a package
ken update [package] Update installed package(s) to latest
ken package Generate ken.yaml manifest from local content
ken publish Publish packages to the registry (requires gh CLI)

Package format

Packages are identified as author/package (e.g. diamondBelema/nucleic-acid). Each package includes a ken.yaml manifest:

ken.yaml
id: diamondBelema/nucleic-acid
name: Nucleic Acid
version: 1.0.1
author: diamondBelema
description: "DNA and RNA structure, replication, transcription, translation"
subjects:
  - nucleic-acid
tags: [biochemistry, molecular-biology]

See the registry for available packages.

Data architecture

ken separates content from state:

  • Content (read-only): ~/Documents/learn/subjects/<subject>/ — concepts, flashcards, quizzes, notes, groups. ken never writes to this directory.
  • State (writable): ~/.local/share/ken/ (Linux) / ~/AppData/Local/ken/ (Windows) — per-subject progress JSON, registry state, stats.

This means:

  • Git repos containing course materials stay clean (no progress files)
  • Multiple users can study the same content independently
  • Reinstalling or updating content never touches progress

Diagrams

ken supports two diagram types, both viewable with d in flashcards, quiz, and map views:

  • External SVG: Reference an .svg or .png file — ken opens it with your system viewer
  • Mermaid: Embed mermaid diagrams inline in your concept YAML — ken renders them to ASCII in the terminal or SVG in your system viewer
